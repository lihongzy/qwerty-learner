# Qwerty Learner 高级感双主题设计规范

## 目的
这份文档定义 Qwerty Learner 的统一视觉主题。

后续所有页面、组件、弹窗、按钮、设置面板、卡片、图表和交互样式，都应以这份规范为设计基线。

目标不是“做得花”，而是做出这三个特征：
- 高级感
- 克制但有记忆点
- 白天与夜晚主题都完整成立，而不是简单反色

## 设计定位
主题方向：`Premium Editorial Study Desk`

关键词：
- 白天主题：纸张、奶油、铜橙、雾青、柔光、玻璃感、静谧、精致
- 夜晚主题：深墨、炭灰、暖金、冷青、低照度、安静、沉浸、克制奢华

整体气质：
- 不走常见紫色渐变科技风
- 不走默认 Tailwind 灰白后台风
- 不走二次元糖果色
- 更接近“高级文具 + 书桌灯光 + 编辑部排版”的感觉

视觉记忆点：
- 暖铜橙与冷雾青形成对照
- 卡片像半透明磨砂纸片
- 页面背景带轻纹理和柔和光斑
- 强调信息通过质感和层次表达，不靠大红大绿

## 主题原则

### 1. 永远优先层次，不优先颜色数量
配色不追求多，而追求主次明确：
- 背景层
- 表面层
- 边框层
- 文字层
- 强调层

### 2. 白天和夜晚是两套完整空间，不是简单反转
白天主题要像纸面和自然光。
夜晚主题要像桌灯下的深色工作台。

### 3. 强调色只做“点睛”
强调色主要用于：
- 主按钮
- 当前选中态
- 重要图标
- 进度
- 焦点状态

不要让强调色铺满整页。

### 4. 全局尽量柔和，局部允许锋利
整体边角和阴影可以柔和，但：
- 当前激活项
- 重要 CTA
- 高优先级反馈
可以更有力量。

## 白天主题规范

### 氛围
像早晨到午后的学习桌面：
- 柔暖底色
- 纸面颗粒感
- 轻雾化高光
- 铜橙按钮
- 冷青辅助点缀

### 白天主题色板
推荐基础变量：

```css
:root {
  --app-bg: #f7f1e8;
  --app-bg-soft: #fbf8f1;
  --app-surface: rgba(255, 253, 249, 0.82);
  --app-surface-strong: rgba(255, 251, 245, 0.94);
  --app-border: rgba(120, 92, 52, 0.16);
  --app-shadow:
    0 30px 60px rgba(74, 53, 26, 0.12),
    0 12px 24px rgba(74, 53, 26, 0.08);
  --app-shadow-soft:
    0 18px 40px rgba(74, 53, 26, 0.08),
    0 8px 18px rgba(74, 53, 26, 0.05);
  --app-text: #2a2017;
  --app-text-muted: #6d5a47;
  --app-accent: #c8682b;
  --app-accent-strong: #a94d1b;
  --app-accent-cool: #456b73;
}
```

### 白天背景
必须具备：
- 大面积暖色底
- 至少两层柔和 radial-gradient
- 一层很轻的纹理或网格

不能：
- 纯白平铺
- 默认浅灰底
- 高饱和大面积彩色背景

### 白天卡片
推荐表现：
- 半透明浅色表面
- 细边框
- 模糊背景
- 阴影柔和且纵深明显

建议：
- 圆角 `16px` 到 `24px`
- 卡片边框比背景略深
- 不要使用太黑的阴影

## 夜晚主题规范

### 氛围
像深夜桌灯下的专注练习：
- 深墨底
- 低亮暖光
- 控制对比
- 明亮但不刺眼

### 夜晚主题色板
推荐基础变量：

```css
html.dark {
  --app-bg: #11161d;
  --app-bg-soft: #161c25;
  --app-surface: rgba(25, 33, 44, 0.82);
  --app-surface-strong: rgba(31, 41, 55, 0.92);
  --app-border: rgba(255, 225, 189, 0.08);
  --app-shadow:
    0 32px 80px rgba(0, 0, 0, 0.36),
    0 14px 30px rgba(0, 0, 0, 0.22);
  --app-shadow-soft:
    0 18px 44px rgba(0, 0, 0, 0.24),
    0 10px 24px rgba(0, 0, 0, 0.16);
  --app-text: #f3eadf;
  --app-text-muted: #c9b79e;
  --app-accent: #e38c4f;
  --app-accent-strong: #f0a96d;
  --app-accent-cool: #84afb4;
}
```

### 夜晚背景
必须具备：
- 深色渐变底，不是纯黑
- 至少一个暖色光斑
- 至少一个冷色辅助光斑

不能：
- 大面积死黑
- 过亮的紫蓝科技风
- 高饱和霓虹色大面积使用

### 夜晚卡片
推荐表现：
- 深色半透明表面
- 比白天更强的阴影层次
- 边框比背景略亮，但保持低对比

## 字体规范

### 全局正文
优先使用：
- `Avenir Next`
- `Segoe UI`
- `PingFang SC`
- `Hiragino Sans GB`
- `Microsoft YaHei`

原则：
- 正文清晰
- 中文可读性优先
- 不使用默认浏览器字体风格作为最终效果

### 标题
标题不需要花哨字体，但要：
- 字重明确
- 间距克制
- 颜色有层次

标题适合：
- `font-semibold`
- `font-bold`
- 字号比正文明显拉开，但不要全站都超大

### 等宽文本
用于：
- 单词
- 键位
- 快捷键
- 数据数字

优先：
- `IBM Plex Mono`
- `SFMono-Regular`
- `Menlo`

## 组件规范

### 1. 按钮
按钮分三层：

#### 主按钮
用途：
- 开始
- 下一步
- 导出
- 提交

表现：
- 铜橙主色渐变
- 有轻微浮起感
- hover 微上移
- 阴影明显但不重

#### 次按钮
用途：
- 次要操作
- 弹窗关闭
- 回退

表现：
- 以表面色为主
- 边框清晰
- 不抢主按钮风头

#### 图标按钮
用途：
- 设置
- 主题切换
- 错题本
- 分析入口

表现：
- 8px 到 12px 内边距
- hover 用浅色背景片衬托
- 不直接裸露图标漂浮

### 2. 卡片
统一规则：
- 大圆角
- 半透明表面
- 细边框
- 纵深阴影
- 内容与背景有明显间距

### 3. Tooltip
Tooltip 必须：
- 使用柔和表面色
- 有细边框
- 有模糊或轻阴影
- 文案简洁

不要：
- 生硬纯黑气泡
- 过小字号
- 过长段落

### 4. Switch
Switch 必须有明显的：
- 关闭底色
- 开启底色
- 可移动 thumb
- hover / focus 反馈

不要只靠文字表达开关状态。

### 5. Slider
Slider 必须：
- 有轨道底色
- 有高亮进度条
- 有明显 thumb
- 拖动时 thumb 层级高于轨道

## 交互规范

### 动效原则
动效应该：
- 短
- 准
- 少

推荐节奏：
- hover：`150ms` 到 `180ms`
- panel / dialog：`200ms` 到 `300ms`
- theme 切换：`300ms`

### 动效类型
优先使用：
- opacity
- translateY
- scale
- shadow
- color

避免：
- 大范围旋转
- 弹跳过猛
- 同时太多元素乱动

## 布局规范

### 页面背景
至少有三层：
1. 主背景渐变
2. 光斑或纹理层
3. 内容表面层

### 间距
整体要“呼吸感明显”：
- 外层容器间距大于内部项间距
- 卡片内上下留白不能过小
- 控件之间对齐优先于挤满空间

### 圆角
统一建议：
- 大容器：`20px` 到 `28px`
- 卡片：`16px` 到 `24px`
- 按钮：`12px` 到 `16px`
- 小控件：`10px` 到 `12px`

## 文案与图标规范

### 文案
整体语言：
- 中文为主
- 品牌名保留英文
- 技术缩写保留英文

按钮文案要短，不要像说明书。

### 图标
图标应：
- 线条统一
- 视觉重量一致
- 尺寸有统一基线

自定义图标 class 统一使用：
- `my-icon`

## 全局 CSS 约束

### 命名规范
项目中自定义 class 必须统一加 `my-` 前缀。

例如：
- `my-card`
- `my-btn-primary`
- `my-tooltip`
- `my-switch-root`
- `my-switch-thumb`
- `my-slider`

禁止继续新增无前缀的自定义 class。

### 变量规范
主题相关颜色、阴影、表面、文字统一走 CSS 变量，不允许把主题核心值散落到几十个组件里。

优先使用：
- `--app-bg`
- `--app-surface`
- `--app-border`
- `--app-shadow`
- `--app-text`
- `--app-text-muted`
- `--app-accent`
- `--app-accent-strong`
- `--app-accent-cool`

## 后续样式修改的执行规则

以后每次改样式，都必须先检查是否符合以下规则：

1. 是否符合这份双主题的气质
2. 是否优先使用主题变量，而不是临时颜色
3. 是否维持白天和夜晚两套完整效果
4. 是否保持高级感、克制感和层次感
5. 是否继续遵守 `my-` 前缀规范

如果新设计和这份规范冲突，以这份规范为准，除非你先更新本文件。

## 推荐工作流

以后你或我在修改样式时，建议按这个流程：

1. 先看本文件
2. 判断当前改动属于：
   - 背景层
   - 表面层
   - 控件层
   - 文案层
   - 动效层
3. 优先复用已有变量和主题逻辑
4. 如果必须新增样式规则，先补到这份文档，再改代码

## 一句话版本
Qwerty Learner 的视觉不再是普通工具站，而是：

`白天像高级纸面学习桌，夜晚像深夜桌灯下的专注工作台。`

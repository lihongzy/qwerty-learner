# 前端界面设计开发文档

## 设计结论

这套界面不应该走“元素很多、设定很满”的主题风，也不应该靠海图、木牌、旧纸去制造所谓氛围。

如果你的审美是 `二次元 + 高级感`，更合适的方向是：

`冷感动画电影风 + 高精度排版 + 克制发光 + 少量锋利装饰 + 强聚焦信息层级`

这不是宅味装饰风，也不是普通学习工具，而是一种更像动画作品片头视觉、角色终端面板、剧场版官网 KV 延展出来的界面系统。

## 不要什么

明确排除下面几类风格：

- 不要海盗感、旧地图感、木质感
- 不要热闹但廉价的热血主题包装
- 不要堆徽章、堆边框、堆世界观文案
- 不要泛二次元紫色霓虹科技风
- 不要萌系、可爱系、贴纸系

你要的不是“有动漫元素”，而是 `像动画视觉总监做出来的界面`。

## 新的审美方向

推荐采用 `冷白银灰 + 深海蓝黑 + 微量绯红 / 琥珀点缀` 的视觉体系。

气质关键词：

- 冷静
- 精致
- 电影感
- 二次元
- 克制
- 锋利
- 透明层次
- 高级材质

参考的不是具体 IP，而是那类高质量动画作品里常见的视觉特征：

- 标题很少，但字体很准
- 画面留白多，但不空
- 色彩压得住，不花
- 局部高光很强，但面积很小
- 组件边缘很干净，层级很明确

## 核心视觉原则

### 1. 不靠“主题道具”建立风格

不要再使用：

- 海图
- 罗盘
- 木牌
- 航线图标化装饰
- 旧纸纹理

新的风格靠这些成立：

- 版式
- 色彩关系
- 透明层叠
- 微弱发光
- 字体权重
- 动效节奏

### 2. 高级感来自克制

高级感不是复杂，而是“每一个视觉动作都有分寸”。

具体表现：

- 主色不超过 3 个
- 高光色只在关键状态使用
- 阴影要深但软，不要脏
- 圆角要统一，不忽大忽小
- 不用廉价渐变，不用彩虹色

### 3. 二次元感来自构图和气氛

不是靠加角色立绘，而是靠：

- 像海报一样的标题布局
- 像片头字幕一样的字重和间距
- 像作战面板一样的状态信息编排
- 像剧场版官网一样的局部辉光和黑场层次

## 配色方案

```css
:root {
  --bg-0: #06070b;
  --bg-1: #0f1117;
  --bg-2: #171a22;
  --panel: rgba(248, 249, 252, 0.06);
  --panel-strong: rgba(248, 249, 252, 0.1);
  --line-soft: rgba(255, 255, 255, 0.08);
  --line-strong: rgba(255, 255, 255, 0.18);
  --text-main: #f5f7fb;
  --text-muted: #a7afc0;
  --text-dim: #737b8c;
  --accent-ice: #c7d2ff;
  --accent-crimson: #d65270;
  --accent-amber: #d7a35b;
  --success: #72c39a;
}
```

配色逻辑：

- 背景不是纯黑，而是有层次的蓝黑灰
- 面板不是白卡，而是低透明雾面玻璃
- 文字以冷白为主，灰蓝为辅
- 点缀色优先绯红和琥珀，不要大面积使用
- 高亮状态可以用极淡冰蓝，但只能小范围出现

## 字体策略

你要的高级感，不适合太花的字体。

建议组合：

- 中文标题：`Noto Serif SC`
- 中文正文：`Noto Sans SC`
- 英文数字：`JetBrains Mono`
- 英文大标题或数字标签：`Bebas Neue`

使用规则：

- 标题用衬线体制造动画海报气质
- 信息区全部回归无衬线，保持清洁
- WPM、时间、数字全部用等宽字体
- 不使用手写体，不使用过度装饰字体

## 背景与空间感

背景应该像动画片头里的黑场，不应该像主题乐园布景。

推荐结构：

1. 最底层：黑蓝渐变
2. 第二层：大面积虚化光斑，位置偏边缘
3. 第三层：极淡网格线或几何切线
4. 最上层：颗粒噪点与暗角

推荐效果：

- 中央区域干净，用于输入聚焦
- 四周有轻微雾感和辉光
- 边缘可出现很淡的斜切几何线，形成锐度

避免：

- 大面积插画背景
- 花哨粒子
- 紫蓝赛博光效
- 复杂图腾和主题纹样

## 页面气质定义

### Typing 页面

定位应是：`主舞台`，不是控制台，不是仪表盘堆砌。

你需要的 Typing 页面应该有这几个特征：

- 中央单词像主角镜头
- 输入框像字幕轨道，不像传统表单
- 上下信息像片头字幕和状态条
- 速度卡片像作品信息栏，而不是后台数据卡

布局建议：

- 顶部只保留必要操作，压低存在感
- 中央单词和输入区绝对主导视觉
- 底部速度区做成一条横向信息带
- 左右辅助区如果有，必须半透明、弱存在

### ResultScreen

定位应是：`结算海报`，不是统计后台。

高级感结果页应该像一张静止海报：

- 中央一个主结论
- 周围三个关键数据
- 错词列表像附录，不要抢主视觉
- 按钮区克制，不要很多图标

结构建议：

- 顶部大标题
- 中央分数或评级
- 下方时间、正确率、WPM 三个数据点
- 最后只保留 2 到 3 个动作按钮

### Speed 区域

你当前的 `Speed` 组件最大的问题，是太像普通统计卡。

它应该改成：

- 一整条细长的数据带
- 背景是低透明深色面板
- 每项数据之间用细竖线切开
- 数值大，标签小，标签用灰蓝色
- 不要白底、不要厚卡片、不要高圆角

也就是说，不是 5 个 InfoBox 拼出来的“卡片区”，而是一条 `作品状态栏`。

## 组件规范

### 按钮

按钮不该像工具按钮，而该像“精致操作件”。

主按钮：

- 深底浅字
- 细边框
- hover 时轻微提亮和边缘发光
- 不用夸张投影

次级按钮：

- 透明底
- 柔和描边
- 强调字重和间距

危险按钮：

- 只用绯红
- 面积要小，别抢画面

### 卡片

卡片统一走 `雾面玻璃 + 深色描边 + 柔光阴影`。

原则：

- 不是白卡
- 不是纸张
- 不是木板
- 不是拟物装饰件

### Tooltip

Tooltip 应像字幕注释。

- 小
- 轻
- 半透明
- 边界清晰
- 出现速度快

### 图标

图标全部走细线性风格。

不要：

- 可爱圆润图标
- 多色图标
- 强拟物图标

## 动效策略

高级感二次元最怕廉价动效。

推荐：

- 淡入
- 位移 4px 到 12px
- 局部光带掠过
- 结果数字延迟出现
- hover 时透明度和描边变化

不要：

- 弹跳
- 旋转
- 大面积缩放
- 发光呼吸过强
- 持续性晃动

## 文案风格

这类风格不适合太中二。

建议文案调性：

- 短
- 冷静
- 准确
- 有一点作品感，但不过戏

例如：

- 开始练习 -> 开始
- 下一章节 -> 下一章
- 重试 -> 重新开始
- 发音设置 -> 发音
- 练习结果 -> 结果
- 错词复习 -> 错词回看

如果你喜欢更有氛围的版本，也应该很克制：

- 开始练习 -> 进入训练
- 下一章节 -> 继续下一章
- 练习结果 -> 本次记录

## 当前项目的具体改造建议

结合当前 `Typing` 页代码，建议优先改这三处：

### 1. 先改 `Speed`

文件：[src/pages/Typing/components/Speed/index.tsx](d:/postgraduate/qwerty-learner/src/pages/Typing/components/Speed/index.tsx)

当前问题：

- 白卡感太重
- 信息密度像后台
- 缺少动画作品式的精致状态条

目标：

- 改成底部悬浮状态带
- 数值更大，标签更轻
- 弱边框，深色半透明背景

### 2. 再改 `WordPanel`

目标：

- 让单词成为唯一主角
- 输入框变成字幕轨道式输入
- 旁边说明信息全部退后

### 3. 最后改 `ResultScreen`

目标：

- 从后台统计弹窗改成结果海报
- 减少图标和杂项按钮干扰
- 把“评级”和“本次记录”做成主视觉

## 最终方向一句话

你要的不是“动漫主题”，而是：

`有动画气质的高级界面，有冷感、有秩序、有精致光感，但不过度花哨。`
